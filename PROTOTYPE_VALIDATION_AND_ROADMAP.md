# 땅콩전사 키우기 — 프로토타입 검증 및 남은 작업

## 1. 첫 실행 검증

1. Unity를 완전히 종료한다.
2. 저장소에서 `git pull`을 실행한다.
3. Unity 6.3 LTS로 프로젝트를 연다.
4. Safe Mode가 나타나면 Console의 첫 번째 빨간 오류부터 확인한다.
5. 컴파일이 완료되면 Play를 누른다.
6. Console에서 아래 두 로그를 확인한다.

```text
[PeanutWarrior Runtime Audit]
PASS · core idle combat, six-tab Canvas UI, global skill AUTO, final ten-stat growth and save bindings are valid.

[PeanutWarrior Feature Audit]
PASS · six-tab icon UI, top stage selector, global skill AUTO, idle boss timer, final growth stats and core progression are active.
```

## 2. 확정 메뉴 구조

### 하단 고정 메뉴 6개

- SKILL
- 장비
- 성장
- 전직
- 펫
- 상점

스테이지 선택은 메인 화면 상단 스테이지 제목 옆의 `선택` 버튼으로만 연다. 설정은 우측 상단 톱니바퀴 버튼으로만 연다. 별도의 모험·임무·빠른 메뉴는 하단에서 제거했다.

메뉴에는 런타임 생성 아이콘과 둥근 카드형 UI를 적용했다. 참고 게임의 정보 구조만 분석했으며 색상, 아이콘 도형, 메뉴 배치와 땅콩 테마는 별도로 구성했다.

## 3. 메인 전투 확인

- 전투 화면 중앙이 UI에 과도하게 가려지지 않는가
- 상단에 캐릭터 HP·MP, 재화와 스테이지 정보가 표시되는가
- 스테이지 제목 옆에 스테이지 선택 버튼이 있는가
- 우측 상단에는 설정 버튼만 있는가
- 하단에 6개 아이콘 메뉴가 고정되는가
- 별도 `SKILL AUTO` 버튼이 모든 스킬을 한 번에 켜고 끄는가
- 자동 균왕 도전 ON에서는 100/100 달성 시 균왕전에 입장하는가
- 자동 균왕 도전 OFF에서는 100/100 이후에도 일반 사냥을 계속하는가
- 검기와 전투 효과가 풀에서 재사용되는가

## 4. 균왕전 확인

- 입장 시 HP와 MP가 최대치로 회복되는가
- 기본 공격과 스킬 쿨타임이 초기화되는가
- 45초 제한시간이 실제 월드 위에 표시되는가
- 수동 회피용 위험 범위와 패턴 공격이 나타나지 않는가
- 균왕 난이도는 HP, 일반 공격 피해와 제한시간으로 결정되는가
- 균왕전 중 다른 스테이지 선택이 차단되는가
- 사망 또는 시간 초과 시 현재 스테이지 0/100으로 복귀하는가
- 승리 시 다음 스테이지로 이동하는가

## 5. 성장 확인

성장 화면에는 아래 10개 항목만 존재해야 한다.

1. 공격력
2. HP
3. HP 회복량
4. MP
5. MP 회복량
6. 치명타
7. 치명타 피해량
8. 경험치 획득량 증가
9. 골드 획득량 증가
10. 장비 강화 획득량 증가

확인 항목:

- ×1, ×10, ×100 강화가 각 단계의 누적 비용을 정확하게 소모하는가
- 치명타가 100%에서 MAX로 표시되고 더 이상 강화되지 않는가
- 몬스터 처치와 균왕 처치로 경험치가 증가하는가
- 경험치 요구량을 채우면 캐릭터 레벨이 증가하는가
- 장비 강화 획득량 증가가 장비 강화 재료 획득에 적용되는가
- 골드 획득량 증가와 HP 회복량이 실제 전투에 적용되는가

## 6. SKILL 확인

- 개별 스킬 AUTO 버튼이 존재하지 않는가
- 메인 화면과 SKILL 화면의 전체 AUTO 버튼 상태가 일치하는가
- AUTO OFF에서는 사냥·균왕 스킬이 모두 자동 발동하지 않는가
- AUTO ON에서는 현재 전투 모드의 스킬이 자동 발동하는가
- 기존 8개 스킬 레벨과 강화 데이터가 유지되는가
- 상세 스킬 구성과 장착 방식은 다음 재설계 전까지 확장하지 않는가

## 7. 장비 확인

- 무속성·화염·냉기·번개 검이 각각 독립적으로 저장되는가
- 등급 순서가 `레어 → 에픽 → 유니크 → 레전드`인가
- 기존 숫자 등급 저장 데이터가 새 명칭으로 정상 표시되는가
- 동일 등급 검 3개가 다음 등급으로 합성되는가
- 사냥 장비와 균왕 장비를 별도로 선택할 수 있는가
- 장비 상세 옵션과 슬롯 구조는 다음 재설계 전까지 추가하지 않는가

## 8. 전직·펫·상점 확인

- 전직이 하단 독립 메뉴로 열리는가
- 2차 전직에서 펫 3슬롯이 해금되는가
- 펫 3마리가 실제 2D 월드에 표시되고 자동 공격하는가
- 펫은 사망하거나 공격 대상이 되지 않는가
- 펫 알 구매와 부화가 작동하는가
- 오늘의 보급은 하루 한 번만 지급되는가
- 검 소환 결과가 레어·에픽·유니크·레전드로 표시되는가

## 9. 스테이지·보상·저장 확인

- 상단 `선택` 버튼에서 월드별 1~30 스테이지가 표시되는가
- 미해금 스테이지 버튼이 비활성화되는가
- 해금된 스테이지 이동 시 전투장과 처치 수가 초기화되는가
- 최초 균왕 클리어 보상이 한 번만 지급되는가
- 골드·조각·다이아의 음수 값과 잘못된 스테이지 범위가 자동 보정되는가
- 30초마다 핵심 진행 데이터 백업이 생성되는가
- 기본 오프라인 보상과 전투력·최고 스테이지 보정이 중복 없이 지급되는가

## 10. 현재 프로토타입 이후 남은 작업

### 우선 설계 보류

- 검의 상세 종류, 슬롯, 옵션과 강화 방식
- SKILL 종류, 장착 수, 등급, 획득 방식과 밸런스

위 두 항목은 기획 확정 전까지 현재 데이터 유지와 기본 화면만 사용한다.

### 실제 게임 디자인

- 땅콩전사 캐릭터 원화 및 전직별 스프라이트
- 월드별 몬스터·균왕 디자인
- 걷기·공격·스킬·피격·사망 애니메이션
- 월드별 타일맵과 배경 오브젝트
- 정식 UI 아이콘 세트와 메뉴 배경
- 전투 파티클, 효과음, 배경음, UI 사운드

### 코드 구조 정식화

- 리플렉션 기반 연결을 명시적 인터페이스와 컴포넌트 참조로 교체
- 전투·성장·장비·스킬·펫 서비스를 분리
- 밸런스 수치를 ScriptableObject로 이전
- 런타임 생성 UI와 월드를 Prefab·Scene 구조로 이전
- PlayerPrefs를 버전 관리 가능한 정식 저장 파일로 교체
- 몬스터·피해 숫자·사망 파편까지 Object Pool 적용

### 모바일 출시 준비

- 다양한 Android 해상도와 노치 기기 테스트
- 저사양 기기 성능 및 발열 테스트
- 터치 영역과 글자 크기 접근성 검토
- Android App Bundle, 서명 키, 스토어 테스트 트랙
- iOS Xcode 프로젝트, 인증서, 프로비저닝
- 앱 아이콘, 시작 화면, 스토어 스크린샷

## 11. 검증 상태 표기 원칙

- **GitHub 반영 완료**: 코드가 저장소에 커밋된 상태
- **정적 검토 완료**: 파일·타입·메서드 연결을 코드 기준으로 검토한 상태
- **Unity 검증 완료**: 실제 Unity 에디터에서 컴파일과 Play Mode를 확인한 상태
- **실기기 검증 완료**: Android/iOS 장치에서 설치 및 조작을 확인한 상태

현재 자동 검사는 Unity에서 Play 또는 Test Runner를 실행해야 결과를 확인할 수 있다.
