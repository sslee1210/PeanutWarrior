# 땅콩전사 키우기 — 프로토타입 검증 및 남은 작업

## 1. 첫 실행 검증

1. Unity를 완전히 종료한다.
2. 저장소에서 `git pull`을 실행한다.
3. Unity 6.3 LTS로 프로젝트를 연다.
4. Safe Mode가 나타나면 Console의 첫 번째 빨간 오류부터 확인한다.
5. 컴파일이 완료되면 Play를 누른다.
6. Console에서 아래 두 로그를 확인한다.

```text
[PeanutWarrior Runtime Audit]
PASS · core combat, progression, Canvas UI and reflection bindings are valid.

[PeanutWarrior Feature Audit]
PASS · Canvas, world theme, boss warnings, effects, sword progression and balance layers are active.
```

## 2. 메인 전투 확인

- 실제 2D 땅콩밭 배경이 표시되는가
- 플레이어와 최대 24마리의 몬스터가 자유 이동하는가
- 플레이어와 몬스터가 인식 범위 안에서만 서로 접근하는가
- 기본 공격과 돌진 베기가 발동하는가
- 일반 공격 및 검술 사용 시 검기·원형 파동 효과가 보이는가
- 몬스터가 서로 완전히 겹치지 않는가
- 100/100 이후에도 몬스터가 계속 등장하고 재화를 지급하는가
- 자동 균왕 도전 OFF에서는 계속 사냥하는가
- 자동 균왕 도전 ON에서는 100/100 달성 시 균왕전에 입장하는가

## 3. 균왕전 확인

- 입장 시 HP와 MP가 최대치로 회복되는가
- 기본 공격, 8개 검술, 돌진 쿨타임이 초기화되는가
- 45초 제한시간과 패턴명이 실제 월드 위에 표시되는가
- 위험 범위가 공격 전에 보이는가
- 광폭화 상태가 표시되는가
- 사망 시 현재 스테이지 0/100으로 복귀하는가
- 승리 시 다음 스테이지로 이동하는가

## 4. 성장·검·검술 확인

- 기초 능력 ×1, ×10, ×100 강화가 정확한 총비용을 소모하는가
- 껍질 단련이 HP와 재생 능력에 반영되는가
- 전직 조건과 전직 결과가 정상인가
- 2차 전직에서 미니 3슬롯이 해금되는가
- 속성 연구가 조각을 소모하는가
- 방치 효율과 최대 방치 시간이 저장되는가
- 검 소환이 일반·희귀·전설·신화 중 하나를 지급하는가
- 동일 등급 검 3개가 다음 등급으로 합성되는가
- 사냥 검과 균왕 검을 별도로 장착할 수 있는가
- 검 등급·레벨·속성 연구 보너스가 실제 피해에 적용되는가
- 8개 검술의 자동 사용 ON/OFF가 실제 전투에 적용되는가

## 5. 미니·임무·상점 확인

- 미니 3마리가 독립 이동 및 자동 공격하는가
- 미니는 사망하거나 공격 대상이 되지 않는가
- 알 구매와 60초 부화가 작동하는가
- 처치·스테이지·성장 보상이 중복 지급되지 않는가
- 오늘의 보급은 하루 한 번만 지급되는가
- 7일 접속 주기가 정상 반복되는가

## 6. 저장·방치 확인

- 즉시 저장 후 Play를 종료한다.
- 다시 Play하여 스테이지, 자원, 강화, 전직, 검, 미니가 유지되는지 확인한다.
- 30초 이상 종료 후 오프라인 골드·조각이 지급되는지 확인한다.
- 기본 보상과 방치 연구 보너스가 구분되어 추가되는지 확인한다.

## 7. Unity 메뉴 도구

```text
Peanut Warrior
├─ Build
│  ├─ Apply Android Settings
│  ├─ Apply iOS Settings
│  └─ Build Android Development APK
└─ Validation
   └─ Run EditMode Tests
```

Android 빌드는 Build Settings에 활성화된 Scene이 있어야 한다. iOS 최종 빌드와 서명에는 macOS 및 Xcode가 필요하다.

## 8. 현재 프로토타입 이후 남은 작업

### 실제 게임 리소스

- 땅콩전사 캐릭터 원화 및 스프라이트 시트
- 전직 단계별 외형
- 걷기, 기본 공격, 돌진, 검술, 피격, 사망 애니메이션
- 월드별 일반 몬스터 및 균왕 이미지
- 실제 배경 타일맵과 오브젝트
- 속성별 파티클 및 검기
- 효과음, 배경음, UI 사운드

### 코드 구조의 정식화

- 리플렉션 기반 프로토타입 연결을 명시적 인터페이스와 컴포넌트 참조로 교체
- 전투·성장·검·미니·미션 서비스를 분리
- ScriptableObject에 모든 밸런스 데이터 이전
- 런타임 자동 생성 오브젝트를 Prefab과 Scene 구성으로 교체
- PlayerPrefs 저장을 버전 관리 가능한 JSON/바이너리 저장으로 교체
- 저장 데이터 마이그레이션과 손상 복구
- Object Pool을 이용한 몬스터·피해 숫자·이펙트 최적화

### 콘텐츠와 밸런스

- 월드별 몬스터 능력치 및 보상표
- 균왕별 고유 패턴과 페이즈
- 검 등급별 세부 옵션과 합성 밸런스
- 검술별 실제 범위·타수·MP·쿨타임 조정
- 미니 종류와 부화 확률, 등급, 도감
- 전직 단계 확장
- 오프라인 보상과 경제 인플레이션 조정
- 일일·주간 임무 및 업적 확장

### 모바일 출시 준비

- 다양한 Android 해상도와 노치 기기 테스트
- 저사양 기기 성능 및 발열 테스트
- 터치 영역과 글자 크기 접근성 검토
- Android App Bundle, 서명 키, 스토어 테스트 트랙
- iOS Xcode 프로젝트, 인증서, 프로비저닝
- 앱 아이콘, 시작 화면, 스토어 스크린샷
- 개인정보 처리방침, 이용약관, 고객지원
- 결제·광고·분석·클라우드 저장은 사업 정책 결정 후 별도 구현

## 9. 검증 상태 표기 원칙

- **GitHub 반영 완료**: 코드가 저장소에 커밋된 상태
- **정적 검토 완료**: 파일·타입·메서드 연결을 코드 기준으로 검토한 상태
- **Unity 검증 완료**: 실제 Unity 에디터에서 컴파일과 Play Mode를 확인한 상태
- **실기기 검증 완료**: Android/iOS 장치에서 설치 및 조작을 확인한 상태

현재 자동 검사는 Unity에서 Play 또는 Test Runner를 실행해야 결과를 확인할 수 있다.
