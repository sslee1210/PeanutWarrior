# 땅콩전사 키우기 — 구현 상태 및 검증 순서

게임 규칙의 최상위 기준은 저장소 루트의 [`GAME_DESIGN_SPEC.md`](GAME_DESIGN_SPEC.md)다.

## 1. 이번 구현 완료 범위

SKILL 상세 구성과 장비/검 상세 구성을 제외한 코드 기능을 완료했다.

### 진행 및 전투

- 무한 스테이지
- 월드당 30스테이지
- 일반 몬스터 100마리 처치 후 보스 도전
- 보스 자동 도전 기본 ON
- 자동 OFF에서는 100/100 상태로 일반 사냥 지속
- 보스 입장 시 HP·MP·공격 쿨타임 초기화
- 보스 45초 제한시간
- 수동 회피 패턴 없음
- 보스 승리 시 다음 스테이지 이동
- 보스 실패 시 현재 스테이지 0/100
- 일반 사냥 사망 시 이전 스테이지 0/100
- 보스전 중 스테이지 이동 차단
- 스테이지별 최초 클리어 보상

### 성장

성장 항목은 아래 10개만 사용한다.

1. 공격력
2. HP
3. HP 회복량
4. MP
5. MP 회복량
6. 치명타
7. 치명타 피해량
8. 경험치 획득량 증가
9. 골드 획득량 증가
10. 장비 강화 획득량 증가

- ×1, ×10, ×100 강화
- 치명타 100% MAX 제한
- 일반 몬스터·보스 경험치
- 골드 추가 획득 적용
- HP 자동 회복 적용
- 장비 강화 재료 획득 적용
- 방치 경험치와 장비 강화 재료 적용

### 전직

8단계 전직이 구현되어 있다.

1. 새싹 껍질
2. 전투 껍질
3. 황금 수호 껍질
4. 화염 갑각
5. 빙결 갑각
6. 뇌광 갑각
7. 왕실 땅콩 갑주
8. 차원 수호자

- 스테이지·전투력·골드·다이아 조건
- 전직별 능력치 배율
- 전직별 기본 공격 타수
- 전직 시 HP·MP 완전 회복
- 2차 전직 시 펫 3슬롯 해금
- 전직별 임시 색상·크기·오라 표시
- 기존 2단계 저장 데이터 마이그레이션

### 펫

- 화염 `불씨 땅콩`
- 냉기 `서리 땅콩`
- 번개 `번개 땅콩`
- 3슬롯 고정
- 독립 이동·자동 공격
- 무적·공격 대상 제외
- 펫별 레벨
- 1~5성 승급
- 중복 부화 조각
- 누적 부화 도감
- 펫 성장과 기존 전투 수치 연결
- 기존 미니 부화 기록 자동 마이그레이션
- 실제 전투 화면에 이름·레벨·별 표시

### 방치 보상

- 최대 8시간
- 최고 해금 스테이지·전투력 효율 반영
- 골드·조각 보상
- 경험치·장비 강화 재료 보상
- 동일 종료 시각 중복 지급 방지

### 상점

장비/검 상품을 제외한 기본 상점 기능:

- 오늘의 접속 보상
- 펫 알 구매
- 성장 골드 보급
- 부화 즉시 완료

실제 결제와 광고는 운영 정책 확정 전까지 포함하지 않는다.

### 설정

- BGM 음량
- 효과음 음량
- 진동 ON/OFF
- 60 FPS
- 30 FPS 절전
- 즉시 저장
- 백업 복구
- 재실행 후 설정 유지

### 저장

- JSON 저장 스키마 v3
- 10초 자동 저장
- 30초 백업
- 앱 백그라운드·종료 시 저장
- 기존 PlayerPrefs 진행 데이터 마이그레이션
- 음수 재화·잘못된 스테이지·전직 범위 보정
- 주 저장 손상 시 백업 복구
- 성장·전직·펫·상점·설정·최초 클리어 저장

### 월드

8개 기본 월드 테마와 보스 이름이 구현되어 있다.

- 땅콩밭 침공
- 곰팡이 창고
- 포식자의 숲
- 얼어붙은 저장고
- 불타는 이세계
- 폭풍의 공중정원
- 황금 껍질 왕국
- 차원 균열 중심부

8개 이후에는 강화·초월 순환 월드로 이어진다.

## 2. 설계 보류 범위

사용자 확정 전까지 더 확장하지 않는다.

### SKILL

- 스킬 종류
- 장착 슬롯
- 등급·획득·진화
- 사냥·보스 프리셋
- 세부 밸런스

현재 확정 사항은 하나의 `전체 SKILL AUTO` 버튼만 사용하는 것이다.

### 장비/검

- 상세 장비 부위
- 옵션
- 강화·합성 방식
- 소환 확률·천장
- 도감 효과

현재 확정 사항:

- 무속성·화염·냉기·번개
- 레어 → 에픽 → 유니크 → 레전드
- 사냥 장착·보스 장착 분리

## 3. Unity 확인 순서

1. Unity를 완전히 종료한다.
2. `git pull`을 실행한다.
3. Unity 6.3 LTS로 프로젝트를 연다.
4. Safe Mode가 나타나면 Console 첫 번째 빨간 오류부터 확인한다.
5. 컴파일 완료 후 Play를 실행한다.
6. 성장·전직·펫·상점·설정 화면을 차례대로 연다.
7. 저장 후 Play를 종료하고 다시 실행해 값이 유지되는지 확인한다.

정상이라면 Console에 다음 로그가 표시된다.

```text
[PeanutWarrior Core Completion Audit]
PASS · advancement, pets, stage rules, idle boss, growth, offline rewards, shop, settings, JSON save and core menu V3 are active. SKILL and equipment remain intentionally deferred.
```

- 일반 몬스터·보스·펫 정식 스프라이트
- 걷기·공격·피격·사망 애니메이션
- 월드별 타일맵·배경 오브젝트
- 정식 UI 아이콘·버튼·패널 리소스
- 전투 파티클·피해 숫자·보상 연출
- 효과음·BGM·UI 사운드 실제 음원
- Android 실기기 성능·발열·해상도 테스트
- iOS Xcode 빌드·서명
- 앱 아이콘·시작 화면·스토어 이미지
- 광고·인앱결제·계정·클라우드 저장 정책

## 5. 상태 표기 원칙

- **GitHub 반영 완료**: 코드가 저장소에 커밋된 상태
- **정적 검토 완료**: 파일·필드·메서드 연결을 코드 기준으로 확인한 상태
- **Unity 검증 완료**: Unity 에디터에서 실제 컴파일과 Play Mode 확인 완료
- **실기기 검증 완료**: Android/iOS 기기에서 설치·조작 확인 완료

현재 자동 검사는 Unity에서 Play 또는 Test Runner를 실행해야 실제 결과를 확인할 수 있다.
